1. Definition

1.1. General

A webpage is a single document or file that is displayed in a web browser when someone visits a specific URL. It contains various elements such as text, images, and links that are presented in a specific layout.
A website, on the other hand, is a collection of webpages that are linked together and accessed through a common domain name or URL. It can contain multiple webpages, including a homepage, about page, contact page, and other pages that provide information, services, or products.
A website can also have other elements such as a navigation menu, footer, header, and sidebars that help to organize the content and make it easier for users to navigate. Additionally, a website can contain interactive elements such as forms, videos, and animations.
In summary, a webpage is a single document that is displayed in a web browser, while a website is a collection of webpages that are linked together and accessed through a common domain name or URL.

1.2. Syntax: tags, elements, attributes

In HTML, tags, elements, and attributes are used to create and define the structure and content of a webpage.

Tags: Tags are HTML elements that are used to define the structure of the webpage. They are enclosed in angle brackets (<>) and can be either opening tags or closing tags. Opening tags are used to start an HTML element, while closing tags are used to end it.

Elements: Elements are made up of tags, content, and attributes. They define the structure and content of a webpage. For example, the HTML element <h1> defines a heading element, and the content between the opening and closing tags is the text that appears as the heading on the webpage.

Attributes: Attributes provide additional information about an HTML element. They are added to the opening tag of an element and are made up of a name and a value. For example, the "src" attribute in the <img> element is used to specify the URL of the image file to be displayed on the webpage.

The <img> element has an "src" attribute that specifies the URL of the image file to be displayed, and an "alt" attribute that provides alternative text in case the image cannot be displayed. The <a> element has an "href" attribute that specifies the URL of the webpage to be linked to.

1.3. Content structuring elements

HTML provides several content structuring elements that are used to create a well-organized and semantically meaningful webpage. These elements help to define the structure of the content and provide information about its meaning, making it easier for search engines, screen readers, and other assistive technologies to interpret the content. Some of the most commonly used content structuring elements in HTML include:

The <header> element is used to define the top section of a webpage, which typically includes a logo, navigation menu, and other elements that are repeated across multiple pages of the website.

The <nav> element is used to define a section of a webpage that contains links to other pages on the website or to external websites.

The <main> element is used to define the main content of a webpage, which typically includes the primary content of the page.

The <section> element is used to define a section of a webpage that contains related content, such as a group of articles or a list of products.

The <article> element is used to define a standalone piece of content that can be read independently, such as a news article, blog post, or product review.

The <aside> element is used to define a section of a webpage that contains content that is related to the main content, but is not essential to the overall meaning of the page

The <footer> element is used to define the bottom section of a webpage, which typically includes copyright information, contact information, and other elements that are repeated across multiple pages of the website.

By using these content structuring elements in HTML, you can create a well-organized and semantically meaningful webpage that is easy to navigate and understand.

2. Text "formatting" elements

2.1. Text "formatting" elements

HTML provides several text formatting elements that allow you to change the appearance of text on a webpage. These elements are used to add emphasis, headings, lists, and other formatting to the text content. Here are some of the most commonly used text formatting elements in HTML:

The <h1>-<h6> elements are used to define headings of varying sizes and importance, with <h1> being the largest and most important. Headings provide a visual hierarchy to the content and help users to quickly identify the main sections of the page.

The <p> element is used to define paragraphs of text. It is the most common text element in HTML.

The <strong> and <em> elements are used to add emphasis to text. <strong> is used to indicate important text that should be bolded, while <em> is used to indicate emphasized text that should be italicized.

The <u>, <s>, and <del> elements are used to underline, strike through, and add a line through text, respectively.

The <blockquote> element is used to quote text from another source. It is often used to format longer quotes and distinguish them from the rest of the text.

The <ul> and <ol> elements are used to create unordered and ordered lists, respectively. Items in an unordered list are preceded by bullet points, while items in an ordered list are numbered.

The <br> element is used to create a line break, while the <hr> element is used to create a horizontal line that can be used to separate sections of content.

By using these text formatting elements in HTML, you can create a webpage that is easy to read and visually appealing. However, it's important to use text formatting elements appropriately and sparingly, as overuse can make the content difficult to read and understand.

2.2. Links

Links are an important feature of HTML that allow you to create hyperlinks to other webpages, documents, or resources on the internet. They allow users to navigate between webpages and access additional content related to the current webpage.
To create a link in an HTML document, you need to use the <a> element, which stands for "anchor." The <a> element requires a minimum of two attributes: href and text.

There are several other attributes that you can use with the <a> element to control the behavior and appearance of the link. Some common attributes include:

target: Specifies where the linked document should open (e.g., in a new window or tab). title: Specifies a tooltip text that appears when the user hovers over the link. rel: Specifies the relationship between the current document and the linked document (e.g., whether it is a bookmark or a reference).

3. Lists, Tables, Forms

3.1. List items

In HTML, list items are created using the <ul> (unordered list) or <ol> (ordered list) element, which contain one or more <li> (list item) elements. Unordered lists display list items with bullet points, while ordered lists display them with numbered or lettered labels. List items can also contain other HTML elements such as text, images, or links.

3.3. Tables

In HTML, tables are created using the <table> element, which contains one or more <tr> (table row) elements. Each table row contains one or more <td> (table data) elements that represent individual cells within the table. Tables can also have header cells using the <th> (table header) element, which are typically displayed in bold and centered. Table cells can contain any type of content, including text, images, links, or other HTML elements. Additionally, tables can be styled using CSS to control their appearance and layout.

3.3. Forms

In HTML, forms are created using the <form> element, which contains various types of input fields that allow users to submit data to a web server. Input fields can include text fields, checkboxes, radio buttons, drop-down menus, and buttons. Each input field is created using a specific type of 'input' element, with additional attributes used to control its behavior and appearance. When a user submits a form, the data is typically sent to a web server using the HTTP POST or GET method, where it can be processed and stored. Forms can also be styled using CSS to control their appearance and layout.
